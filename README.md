Scheduler for python scripts on the Ubuntu server with sending logs to Telegram

Отправьте боту любое текстовое сообщение. Если бот добавлен в группу, отправьте сообщение прямо в этой группе.

Откройте любой браузер и перейдите по следующей ссылке, подставив свой токен:

На экране отобразится строка в формате JSON. 
Найдите в тексте блок "chat".

Для личного чата это будет просто число (например, 456789123).
Для группы или супергруппы ID будет начинаться со знака минус (например, -100987654321).

Шаг 2. Настройка автоматического запуска в Ubuntu (Cron)

Чтобы скрипт работал как полноценный планировщик, доверим его вызов демону cron, который встроен в Ubuntu по умолчанию и работает 24/7.

    Откройте редактор задач cron для вашего пользователя:
    Bash

    crontab -e

    (Если предложит выбрать редактор, выбирайте nano — он самый простой).

    Добавьте в самый конец файла строку с правилом запуска. Например, чтобы мониторинг запускался каждый час:

Plaintext

   0 * * * * /usr/bin/python3 /path/to/monitor.py

Примеры других интервалов:

    */15 * * * * ... — каждые 15 минут.

    0 9 * * * ... — каждый день ровно в 09:00 утра.

    0 0 * * 1 ... — каждый понедельник в полночь.

    Сохраните файл (в Nano: Ctrl + O, затем Enter, и выход Ctrl + X).

Альтернатива: Если "планировщик" должен быть бесконечным Python-процессом

Если вы принципиально не хотите использовать системный Cron, а планировщик должен крутиться как постоянный демон на Python, код monitor.py можно обернуть в библиотеку schedule.

    Установите библиотеку: pip install schedule

    Измените блок запуска в коде:

Python

import schedule
import time

# ... (весь предыдущий код функций остается прежним) ...

if __name__ == "__main__":
    # Пример: запускать каждый день в 12:00
    schedule.every().day.at("12:00").do(run_and_monitor)
    
    # Или пример каждые 30 минут:
    # schedule.every(30).minutes.do(run_and_monitor)

    print("Планировщик запущен и ожидает задач...")
    while True:
        schedule.run_pending()
        time.sleep(1)

    ⚠️ Важно: Если вы выберете этот путь (бесконечный цикл), запускать этот скрипт на Ubuntu Server нужно будет в фоне — например, через утилиту screen, nohup или настроив под него простой systemd сервис, иначе он закроется сразу, как только вы отключитесь от SSH-терминала. Использование Cron (Шаг 2) лишено этих проблем и гораздо надежнее для серверов.

Перед вами откроется точно такой же терминал, но теперь он защищен.

Запустите ваш скрипт:
Bash

python3 server_scheduler.py

Отключитесь от сессии (Detach):
Нажмите комбинацию клавиш: Ctrl + A, а затем сразу клавишу D.
Вы вернетесь в свой основной терминал SSH, а сверху появится надпись [detached]. Теперь можно закрывать консоль — скрипт работает.

Как вернуться и проверить консоль позже:
Когда вы снова подключитесь к серверу по SSH, введите:
Bash

screen -r monitor

Вы снова увидите вывод своего скрипта. Если скрипт уже отработал, сессия закроется сама.
