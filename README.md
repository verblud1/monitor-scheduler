# Monitor-Scheduler

Простой и надежный планировщик для Python-скриптов на сервере Ubuntu с автоматической отправкой логов выполнения в Telegram.

## 🚀 Возможности

* Автоматический запуск целевого скрипта по расписанию.
* Перехват консольного вывода (`stdout` и `stderr`).
* Отправка красивых HTML-отчетов в Telegram.
* Автоматическая обрезка логов под лимиты Telegram (макс. 3500 символов), чтобы сообщения не терялись.
* Защита от сбоев: если Telegram не принимает HTML-разметку, скрипт автоматически отправит лог в сыром текстовом виде.

## 🛠 Шаг 1. Подготовка и настройка скрипта

### 1. Получение Telegram Chat ID и Токена

1. Найдите своего бота в Telegram и нажмите кнопку **Запустить** (`/start`).
2. Отправьте боту любое текстовое сообщение. Если бот добавлен в группу, отправьте сообщение прямо в эту группу.
3. Скопируйте **Токен** вашего бота, который выдал вам `@BotFather` при создании (выглядит как `123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ`).
4. Откройте любой браузер и перейдите по ссылке, подставив свой токен:
`[https://api.telegram.org/bot](https://api.telegram.org/bot)<ВАШ_ТОКЕН_БОТА>/getUpdates`
5. На экране отобразится строка в формате JSON. Найдите блок `"chat"`. Значение параметра `"id"` — это ваш Chat ID:
* Для личного чата это просто число (например, `456789123`).
* Для группы или супергруппы ID начинается со знака минус (например, `-100987654321`).



### 2. Установка зависимостей на сервере

Подключитесь к серверу по SSH, перейдите в директорию проекта и настройте окружение:

```bash
cd FOLDER_WITH_YOUR_PROJECT
apt update && apt install python3.12-venv python3 -y
python3 -m venv .venv
source .venv/bin/activate
pip install requests

```

### 3. Настройка кода скрипта

Обязательно заполните переменные `TOKEN`, `CHAT_ID` и проверьте пути в `TARGET_SCRIPT`.

> ⚠️ **Важно:** В переменной `TARGET_SCRIPT` обязательно указывайте **абсолютные пути**. Окружение Cron и фоновые демоны не умеют работать с относительными путями вида `python main.py`.


## ⚙️ Шаг 2. Настройка расписания (Варианты запуска)

Выберите **один** из двух способов автоматизации, который лучше подходит под ваши задачи.

### Вариант А. Нативный запуск через системный Cron (Рекомендуется)

Этот способ наиболее надежен для серверов, так как запуск контролирует встроенный демон Ubuntu, работающий 24/7. Скрипт отрабатывает и сразу завершает работу, не занимая оперативную память вхолостую.

1. Откройте редактор задач cron:
```bash
crontab -e

```


*(Если система предложит выбрать редактор, выбирайте `nano` — он самый простой).*
2. Добавьте в самый конец файла строку с правилом запуска. Например, для запуска **каждый час**:

```text
   0 * * * * /root/Xiaomi-Unlock-Ultra/.venv/bin/python3 /root/Xiaomi-Unlock-Ultra/server_scheduler.py

```

3. **Примеры других интервалов:**
* `*/15 * * * * ...` — каждые 15 минут.
* `0 9 * * * ...` — каждый день ровно в 09:00 утра.
* `0 0 * * 1 ...` — каждый понедельник в полночь.


4. Сохраните файл: в Nano нажмите `Ctrl + O`, затем `Enter`, и закройте редактор через `Ctrl + X`.

---

### Вариант Б. Бесконечный Python-процесс в утилите Screen

Если вы хотите использовать библиотеку `schedule` внутри Python (без Cron), скрипт превратится в постоянный фоновый процесс.

1. Установите библиотеку `schedule` в ваше виртуальное окружение:

```bash
   pip install schedule

```

2. Измените блок `__main__` в конце вашего скрипта `server_scheduler.py`:

```python
   import schedule
   import time

   # ... (весь остальной код функций остается прежним) ...

   if __name__ == "__main__":
       # Пример: запускать каждый день в 12:00
       schedule.every().day.at("12:00").do(run_and_monitor)
       
       # Или пример каждые 30 минут:
       # schedule.every(30).minutes.do(run_and_monitor)

       print("Планировщик запущен и ожидает задач...")
       while True:
           schedule.run_pending()
           time.sleep(1)

```

3. **Запуск в фоновом режиме через Screen:**
Если запустить такой скрипт напрямую, он закроется сразу, как только вы отключитесь от SSH-терминала. Чтобы этого не произошло, используем `screen`:

```bash
   # Установка утилиты screen
   sudo apt update && sudo apt install screen -y
   
   # Создание новой защищенной сессии с именем "monitor"
   screen -S monitor
   
   # Активация окружения и запуск бесконечного планировщика
   source .venv/bin/activate
   python3 server_scheduler.py

```

4. **Управление сессией Screen:**
* **Отключиться от сессии (оставив скрипт работать в фоне):** Нажмите комбинацию клавиш `Ctrl + A`, а затем сразу клавишу `D`. Вы вернетесь в основной терминал, а сверху появится надпись `[detached]`. Теперь SSH-консоль можно закрывать.
* **Вернуться и проверить логи:** При следующем подключении к серверу введите:
```bash
screen -r monitor
